Aether One™

Dual Raspberry Pi — Trust Infrastructure & Edge Intelligence

Active · v1.0 · 2026

Aether One yhdistää trust-infrastruktuurin ja edge-sensorit kahdeksi erilliseksi, optimoiduksi yksiköksi. Pi 2 toimii dedikoidun trust-ankkurina (PQC attestation, decision hash audit), Pi 5 edge-laskentana ja sensorien lukijana (MQ-9, AetherCam, KRI-laskenta).

Arkkitehtuuri seuraa TN-002:n DCEIN-konseptia (Duration-Capable Edge Intelligence Node) fyysisellä tasolla: hajautettu, paikallisesti operoiva, attestoitu solmu joka ei riipu pilviyhteydestä.

Arkkitehtuuri

Pi 2 · Trust Server
  • PQC attestation (Dilithium3)
  • Nonce generation
  • Device enrollment
  • Decision hash audit
  • Port 5000 · IP 192.168.1.50
Pi 5 · Edge Node
  • MQ-9 kaasusensori (ADC)
  • AetherCam (S21 IP-stream)
  • TrustCore v1.0 C-kernel
  • KRI + LR laskenta
  • Web UI + Drift Monitor
  • Port 8080
Pi 2 (Trust) ←—— WiFi attestation ——→ Pi 5 (Edge)
  PQC + SHA-256                    Sensors → KRI → UI
  Erillinen laite                  Reaaliaikainen laskenta
  Ei näe sensordataa               Attestoi Pi 2:lta

Miksi kaksi laitetta?

Trust-ankkuri on fyysisesti erillinen. Pi 2 ei näe sensordataa suoraan — se ei ole "judge in its own case". Auditlokit ovat erillisellä laitteella. Rakenne muistuttaa HSM/TPM-jaottelua oikeassa tuotantoinfrastruktuurissa.

Pi 5:n teho tarvitaan edge-laskentaan. Sensorien luku (ADC, I2C, IP-stream), TrustCore v1.0 C-kernel (KRI, LR-D), mahdollinen ML-päättely (32 TOPS suunnitelma). Pi 2 riittää mainiosti kevyeen attestation-tehtävään.
