§ 04Metodologia
BEPP v2 = (D_f^1.2 * D_s^1.0 * D_c^0.8) * (1-R)^1.5
dS/dt = P(t) - lambda*R(t)*S(t) - mu*S(t)^2
T(t) = S(t) * max(0, dS/dt) * (1-R(t))

D_s: FinBIF — liito-orava, palokärki, kalasääski · live
D_f: CORINE 2018 -ruudukko (metsäosuus) + Sentinel-2 NDVI-hajonta (aci-corine-proxy) · live (2026-07-08 lähtien)
D_c: YVA-pipeline 56.3 GW tuulivoima (0.61) · kiinteä esimerkkiarvo, ei live-dataa
R: SYKE inspire_ps suojelualueet (Natura2000 SAC/SPA + valtion/yksityiset LSA), 3x otanta eri offseteilla · live (2026-07-08 lähtien)

Huom (2026-07-08): tama on v0.1-prototyyppi (TN-015: "Pre-development"). BEPP-kaava on toteutettu tarkalleen dokumentoidun mukaisesti. D_s, D_f ja R haetaan nyt oikeasti live-lahteista: D_s FinBIF:sta, D_f kahdesta riippumattomasta lahteesta (SYKE CORINE-ruudukko + Sentinel Hub Statistical API NDVI-hajonta), R SYKE:n suojelualue-WMS:sta (aci-corine-proxy). D_f = 0.6*(1-metsaosuus) + 0.4*min(1, NDVI-stDev/0.30) - dokumentoitu approksimaatio, ei validoitu todellista fragmentaatiomittausta vasten. R = suojellun pinta-alan piste-otantaosuus, kolme erillista 7x7-ruudukkoa eri offseteilla (0.25/0.5/0.75) yhdistettyna otantavirheen vahentamiseksi - edelleen pieni otos (nykyinen alue ~2-4 osumaa/147 pistetta), tulkittava varoen. D_c on ainoa yha kiinnitetty SERIES-taulukon esimerkkiarvo. BEPP-kokonaisarvo heijastaa siis kolmea neljasosaa oikeasta live-datasta.
